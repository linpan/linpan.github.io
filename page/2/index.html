python模块:shelve

发表于 2018-08-01

shelve描述

shelve是python3才有的序列化模块，其最大的优点就是操作十分简单

shelve类似于一个key-value数据库，可以很方便的用来保存Python的内存对象，其内部使用pickle来序列化数据。简单来说，使用者可以将一个列表、字典、或者用户自定义的类实例等任意对象保存到shelve中，下次需要用的时候直接取出来，就是一个Python内存对象，不需要像传统数据库一样，先取出数据，然后用这些数据重新构造一遍所需要的对象.

shelve 用法

class shelve.Shelf(dict, protocol=None, writeback=False)

通过open方法直接使用, 返回一个Shelf类的实例

# shelve_create.py
import shelve

with shelve.open('test_shelf.db') as s:
    s['key1'] = {
        'int': 10,
        'float': 9.5,
        'string': 'Sample data',
    }

参数writeback 为 True 时，对字段内容的修改会writeback到数据库中.
import shelve

integers = [1, 2, 3, 4, 5]

# If you're using Python 2.7, import contextlib and use
# the line:
# with contextlib.closing(shelve.open('shelf-example', 'c')) as shelf:
with shelve.open('shelf-example', 'c') as shelf:
    shelf['ints'] = integers

python __new__ 方法

发表于 2018-08-01

new和init的区别

根据官方文档：

init是当实例对象创建完成后被调用的，然后设置对象属性的一些初始值。

new是在实例创建之前被调用的，因为它的任务就是创建实例然后返回该实例，是个静态方法。

也就是，new在init之前被调用，new的返回值（实例）将传递给init方法的第一个参数，然后init给这个实例设置一些参数。

  • 重载 new 方法，需要返回类的实例

一般情况下，我们不需要重载 __new__ 方法。但在某些情况下，我们想控制实例的创建过程，这时可以通过重载 __new__ 方法来实现。

new的应用场景

官方文档指出new方法的两种用法。

允许继承不可变类型（str，int， tuple）

关于这种也有比较多的例子，网上搜到的例子基本上都属于理论性，实际中用法不太常见。

class Word(str):
'''存储单词的类，定义比较单词的几种方法'''

def __new__(cls, word):
    # 注意我们必须要用到__new__方法，因为str是不可变类型
    # 所以我们必须在创建的时候将它初始化
    if ' ' in word:
        print "Value contains spaces. Truncating to first space."
        word = word[:word.index(' ')] #单词是第一个空格之前的所有字符
    return str.__new__(cls, word)

def flatten(l):
  for el in l:
      if isinstance(el, collections.Iterable) and not isinstance(el, (str, bytes)):
          yield from flatten(el)
      else:
          yield el

或：

flatten = lambda *n: (e for a in n
    for e in (flatten(*a) if isinstance(a, (tuple, list)) else (a,)))

生产递归及鸭子类型的生成器方式：

def flatten(L):
    for item in L:
        try:
            yield from flatten(item)
        except TypeError:
            yield item

list(flatten([[[1, 2, 3], [4, 5]], 6]))

补充: “鸭子类型”
鸭子类型(Duck Typing)的名字来源于鸭子测试: “当看到一只鸟走起来像鸭子、游泳起来像鸭子、叫起来也像鸭子，那么这只鸟就可以被称为鸭子”这句话可以理解成这样，当一个对象A具有了对象B的特定方法，那么我们就可以说对象B也是对象A.(可以理解为近亲关系)

它并不要求严格的继承体系，一个对象只要“看起来像鸭子，走起路来像鸭子”，那它就可以被看做是鸭子。可以看出，鸭子类型给予Python这样的语言以多态性。 但是这种多态的实现完全由程序员来约束强制实现（文档、清晰的代码和测试），并没有语言上的约束（如C++继承和虚函数）

Python的file-like object就是一种鸭子类型。对真正的文件对象，它有一个read()方法，返回其内容。但是，许多对象，只要有read()方法，都被视为 file-like object。许多函数接收的参数就是file-like object，你不一定要传入真正的文件对象，完全可以传入任何实现了read()方法的对象。

python: 工厂函数factory functions

发表于 2018-08-01

工厂方法

a factory method returns (new) objects.

工厂函数

A factory function is basically used to return (new) objects.

简单的工厂函数用法:

def fun(N):
    def action(X):
        return X * N
    return action

返回一个新的对象, 其类型是基于输入决定的。

class Car(object):

    def factory(type):
        if type == "Racecar":
            return Racecar()
        if type == "Van":
            return Van()
        assert 0, "Bad car creation: " + type

    factory = staticmethod(factory)

class Racecar(Car):
    def drive(self): print("Racecar driving.")

class Van(Car):
    def drive(self): print("Van driving.")

# Create object using factory.
obj = Car.factory("Racecar")
obj.drive()

函数字典参数

  1. 在函数传参中经常遇到**kwargs,现在总结常用的用法：

    def foo(x, y):
    print(x, y)

> d = {'x':1, 'y':2}
> foo(**d)
# 1 2

  2. 在py3.5后,也可以嵌套

    >d = {'a': 1}
>{'b': 2, **d}
{'b': 2, 'a': 1}

  3. python3 参数带
    根据PEP 3102的定义描述：
    星号可以以一个参数的形式出现在函数声明中的参数列表中，但星号之后的所有参数都必须有关键字（keyword），这样在函数调用时，星号    之后的所有参数都必须以keyword=value的形式调用，而不能以位置顺序调用。

    def func(arg1, arg2, arg3, *, k1, k2):
    pass
#使用方法 func(1,2,3,k1='key',k2='key2')

星号分开位置参数和强制关键字参数分界表示,使用函数时,星号没有实际意义,不需要给星号参数.

Django ContentTypes 描述概念

django 包含一个用来跟踪所有安装的应用模型的contenttypes 应用,它对当前项目中所有基于Django驱动的model提供了更高层次的抽象接口。
ContentType 是一个Django自动创建的模型，它包含2个字段

app_label：app的名称
model：模型的名字（小写）

通过下表, 一个app的下所有model都会写入表中,统一入口管理。

ContentType实例方法

ContentType.get_object_for_this_type(**kwargs)

对表中记录的模型get查询，返回对象

ContentType.model_class()

获得模型类

# 获得contenttype实例
>>> from django.contrib.contenttypes.models import ContentType
>>> user_type= ContentType.objects.get(app_label="auth", model="user")
# 输出 ContentType: user
>>> user_type.model_class()
# class'django.contrib.auth.models.User'
>>>user_type.get_object_for_this_type(username='Guido')
# User: Guido
应用场景
  • 不需要直接导入或者定义模型类, 直接传参app_lable和model给contentype,直接对model完成检索查询。
  • 您可以将另一个模型与ContentType相关联，作为将其实例绑定到特定模型类的方法，并使用这些方法来访问这些模型类。

ContentType管理器

  1. ContentType.objects.get_for_id()
    通过id寻找ContentType类型，这个跟传统的get方法的区别就是它跟get_for_model共享一个缓存，因此更为推荐。

  2. ContentType.objects.get_for_model()
    通过model或者model的实例来寻找ContentType类型

from django.contrib.auth.models import User
ContentType.objects.get_for_model(User)
# 输出结果 ContentType: user

Generic relations 范型关系

给用户模型添加Contentype,使得用户的模型更好的和其他模型建立关系，就Permission模型, ContentType在模型之间建立了范型关系.

from django.contrib.contenttypes.fields import GenericForeignKey
from django.contrib.contenttypes.models import ContentType
from django.db import models

class TaggedItem(models.Model):
    tag = models.SlugField()
    content_type = models.ForeignKey(ContentType, on_delete=models.CASCADE)
    object_id = models.PositiveIntegerField()
    content_object = GenericForeignKey('content_type', 'object_id')

    def __str__(self):
        return self.tag

ForeignKey指向仅一个模型,GenericForeignKey可以跟多个模型建立关联,需要如下必须的三个过程:

  1. 用户模型的ForeignKey to ContentType, 默认字段名称content_type.

    from django.contrib.auth.models import User
guido = User.objects.get(username='Guido')
t = TaggedItem(content_object=guido, tag='bdfl')
t.save()
t.content_object
# User: Guido

  2. 给模型添加object_id 字段,存储来自关联模型的primary key

    object_id = models.PositiveIntegerField()
  3. 给模型添加GenericForeignKey, 传递上面的2个参数,如果上面的参数名称是默认的（content_type, object_id）可以忽略不传参。

反向 GenericRelation

给模型模型添加 “reverse” generic relationship,使得可以访问关联的模型。
from django.contrib.contenttypes.fields import GenericRelation
from django.db import models

class Bookmark(models.Model):
    url = models.URLField()
    tags = GenericRelation(TaggedItem)

Bookmark 实例有tags属性, 可以检索关联TaggedItems模型的数据.

理解模型中的reserve 反向

django 默认每个主表的对象都有一个是外键的属性，可以通过它来查询到所有属于主表的子表的信息。这个属性的名称默认是以子表的名称小写加上_set()来表示(上面默认以b_set访问)，默认返回的是一个querydict对象。related_name 可以给这个外键定义好一个别的名称

例如B中有一个 models.ForeignKey(A) 。而当我们需要反向查询 A 中某个具体实例所关联的 B 时,可能会用到 A.B_set.all() 或 B.objects.filter(A) 这两种不同的方法

对于用ForeignKey 来定义的关系来说，在关系的另一端也能反向的查询得到相应的结果.
