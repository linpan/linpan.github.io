并行世界的线程

发表于 2018-02-09 | 分类于 python

线程

实现线程的流程

使用threading模块实现一个新的线程，需要下面3步：

  • 覆盖 init(self [,args]) 方法，可以添加额外的参数
  • 最后，需要覆盖 run(self, [,args]) 方法来实现线程要做的事情
  • 当你创建了新的 Thread 子类的时候，你可以实例化这个类，调用 start() 方法来启动它。线程启动之后将会执行 run() 方法。

join（） 阻塞主线程直到所有线程完成。
其实创建一个进程的流程一样的。

线程锁

四种方式实现线程同步

1.Lock锁

  • 锁有两种状态： locked（被某一线程拿到）和unlocked（可用状态）
  • 我们有两个方法来操作锁： acquire() 和 release()
    需要遵循以下规则：先申请，后释放
  • 如果状态是unlocked， 可以调用 acquire() 将状态改为locked
  • 如果状态是locked， acquire() 会被block直到另一线程调用 release() 释放锁
  • 如果状态是unlocked， 调用 release() 将导致 RuntimError 异常
  • 如果状态是locked， 可以调用 release() 将状态改为unlocked

当然锁也有负面作用

尽管理论上行得通，但是锁的策略不仅会导致有害的僵持局面。还会对应用程序的其他方面产生负面影响。这是一种保守的方法，经常会引起不必要的开销，也会限制程序的可扩展性和可读性。更重要的是，有时候需要对多进程共享的内存分配优先级，使用锁可能和这种优先级冲突。最后，从实践的经验来看，使用锁的应用将对debug带来不小的麻烦。所以，最好使用其他可选的方法确保同步读取共享内存，避免竞争条件。

2.RLock锁(重复锁)

如果你想让只有拿到锁的线程才能释放该锁，那么应该使用 RLock() 对象。和 Lock() 对象一样，
RLock() 对象有两个方法： acquire() 和 release() 。当你需要在类外面保证线程安全，又要在类内使用同样方法的时候 RLock() 就很实用了。

  • 谁拿到谁释放。如果线程A拿到锁，线程B无法释放这个锁，只有A可以释放；
    • 同一线程可以多次拿到该锁，即可以acquire多次；
      -
    • acquire多少次就必须release多少次，只有最后一次release才能改变RLock的状态为unlocked）

3.Condition 条件

其中某些线程在等待某一条件发生，其他的线程会在该条件发生的时候进行通知。一旦条件发生，线程会拿到共享资源的唯一权限

4. Event 事件

事件是线程之间用于通讯的对象。有的线程等待信号，有的线程发出信号。基本上事件对象都会维护一个内部变量，可以通过 set() 方法设置为 true ，也可以通过 clear() 方法设置为 false 。 wait() 方法将会阻塞线程，直到内部变量为 true 。

函数

在Cython中有3种函数定义:

  • def: 传入python对象，返回python对象，直接调用
  • cdef: 传入python对象或C/C++值，返回python对象或C/C++值，不可直接调用, 可以使用指针

  • cpdef: 以上两者的混合,Python, Cython 都可以调用，不能使用指针

  • 类中的 cdef 定义C方法，而 cpdef 可以同时定义C方法和Python方法
    只有def和cpdef定义的函数在编译后可以在python代码中直接调用（run-time），cdef定义的函数则不能import 使用。 不过你可以使用def来对外封装

参数转换

如上所示，传递一个值是很简单的，只要稍稍注意一下它的类型。在python和C/C++之间有一些自动的类型转换:
struct 和python 字典类似；char* 代表 字符指针类

+-------------------------------------------------------------------------------------------+
|                C types                          |  From Python types  |  To Python types  |
| [unsigned] char [unsigned] short int, long      |  int, long          |    int            |
| unsigned int unsigned long [unsigned] long long |  int, long          |    long           |
| float, double, long double                      |  int, long, float   |    float          |
| char*   char* 代表 字符指针类                     |  str/bytes          |    str/bytes      |
| struct                                          |                     |    dict           |
+-------------------------------------------------------------------------------------------+

动态添加属性

有两种方式：

cdef class Animal:

cdef int number_of_legs
cdef dict __dict__

def __cinit__(self, int number_of_legs):
    self.number_of_legs = number_of_legs


dog = Animal(4)
dog.has_tail = True

# 属性存在dict里
>>> area.dog.__dict__
{'has_tail': True}

或者：

cdef class Animal:

cdef int number_of_legs

def __cinit__(self, int number_of_legs):
    self.number_of_legs = number_of_legs


class ExtendableAnimal(Animal):  # Note that we use class, not cdef class
    pass


dog = ExtendableAnimal(4)
dog.has_tail = True

cpython 导入

格式声明：

cimport module [, module...]

from module cimport name [as name] [, name [as name] ...]

模块定义在.pxd文件, 在pyx 导入

dishes.pxd:

cdef enum otherstuff:
    sausage, eggs, lettuce

cdef struct spamdish:
    int oz_of_spam
    otherstuff filler
restaurant.pyx:

from __future__ import print_function
cimport dishes
from dishes cimport spamdish

cdef void prepare(spamdish *d):
    d.oz_of_spam = 42
    d.filler = dishes.sausage

def serve():
    cdef spamdish d
    prepare(&d) #  取地址,因为形参是结构体指针, （* 取值） 传地址到 形参
    # 恶补c :array[20] &array是整个数组array的首地址，array是数组首元素的首地址（和&array[0]一样）
    print(f'{d.oz_of_spam} oz spam, filler no. {d.filler}')

其他Py_ssize_t

Py_ssize_t 是python C API的定义类型

Py_ssize_t allocated;  # 是数据类型（比int大），Py_ssize_t 有符号long类型, 长度参数的类型(整型或者 Py_ssize_t)在包含

Python.h 官方解释如下

对Python的C API进行广泛的更改，使用新的 Py_ssize_t类型定义而不是int，将允许解释器在64位平台上处理更多数据。此更改不会影响Python在32位平台上的容量

python操作数据库的规范问题

发表于 2018-02-09 | 分类于 python ， 数据库规范 ， mysql

sql 语句的规范化书写

字段

禁止使用mysql内部保留关键词作为字段名称。例如 更新数据库某个字段为空,必然报错

UPDATE user_preference SET use =’’ WHERE id=’3’

当然在可视化命令执行，use会特别强调处理，解决方案是规范sql语言的书写，尤其用python写mysql语句，

UPDATE user_preference SET ’use‘ =’’ WHERE id=’3’

所以以后凡是字段都在使用``(与单引号差别很大的！)

字段的值

  • 如何给字段赋予空值（‘’）
  • 如何给字段格式加上引号

第一个问题解决是:

>>> query = "update roles set name=%s" % ('')
# 报错：update roles set name=

符合python风格的书写是用双引号包含，这样内部的单引号不会转义了。

query = "update roles set name=%s" % (“‘’”)

第二个问题是占位符%s需要加上引号

>>> query = "update roles set name=‘%s’" # 或者
>>> query = ’update roles set name=“%s”' # 单引号与双引号的区别

引号问题

比如‘hero’s pens’ 在插入、更新中无法引号无法匹配，解决方式（推荐）

字段内部的单引号不转义 需要double ‘’引号

用python替换：

>>> s = "hero's pens".replace("'", "''")
   "hero''s pens"

如果遇到单个引号，字符串要用双引号，你也会明白python 有单引号、双引号存在的意义了。

性能优化

conn.commit() 最好插入、更新多条再提交
