查看编译的镜像

> docker images

启动镜像进入容器

# 普通模式
docker  run  -t  -i  jdk8/centos  /bin/bash
# 可以使用systemctl命令
docker  run  -t  -i  jdk8/centos   /usr/sbin/init
# 给容器别名 指定端口启动
docker run -tid --name eoa    -p 6380:6379 -p 8082:8080 -p 9082:9081 centos7/eoa    /usr/sbin/init

查看允许的容器

> docker ps

退出容器

exit
或者 ctrl+p+q

再次进入允许的容器

如果想进入独立的环境可以使用另一种进入容器的方式：
docker exec -it bash
docker exec -it 8e185ef88c14 bash 或者
docker exec -it 8e bash 镜像id的前2位

文件拷贝

把文件从容器拷贝到宿主机：

docker cp 容器名：要拷贝的文件在容器里面的路径        要拷贝到宿主机的相应路径

把文件从宿主机拷贝到容器：

docker cp 要拷贝的文件路径 容器名：要拷贝到容器里面对应的路径
docker cp   ./nginx-1.12.2.tar.gz   7db885ebfe04:/usr/local/

容器停止、删除、退出

docker stop [NAME]/[CONTAINER ID]:将容器退出。

docker kill [NAME]/[CONTAINER ID] :强制停止一个容器。

docker rm [NAME]/[CONTAINER ID]:不能够删除一个正在运行的容器，会报错。需要先停止容器。

用commit命令创建镜像

进入容器可以继续安装其他的软件、服务及配置等,然后保存提交

要想利用现有的容器制作成镜像：先退出该容器，不要在容器中，然后，

docker commit 当前容器的ID 要保存镜像的名称

如：docker commit 614122c0aabb aoct/apache2

查看镜像信息

使用docker inspect命令查看新创建的镜像的详细信息

docker inspect your-CONTAINER
