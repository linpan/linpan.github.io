eval和exec 的高级用法.md

发表于 2019-11-05 | 分类于 python

eval 的语法

eval 执行一字符串表达式,并返回表达式的值。

有三个参数，表达式字符串，globals变量作用域，locals变量作用域。

globals:  a dict
locals: a mapping object. Dictionary is the standard and commonly used mapping type in Python.

其中第二个和第三个参数是可选的。如果忽略后面两个参数，则 eval 在当前作用域执行。

>>>  a = 1
>>>  eval("a + 1")
2
>>>  eval("a = 2")
Traceback (most recent call last):

如果指定globals参数:

>>>  a = 1
>>>  g = {'a':10}
>>>  eval("a + 1", g)
11

如果指定locals 参数:

>>>  a = 10
>>>  b = 20
>>>  c = 30
>>>  g = {'a':1, 'b':2}
>>>  l = {'b':200, 'c ':300}
>>>  eval('a+b+c', g, l)  // a来自global作用域; b, c来自local作用域
501

eval 给我带来了极大的灵活性，同时也带来了隐含的危险性，当我们使用它们的时候应该总是记得详细指定其执行的作用域

eval 的语法

exec_stmt : “exec” expression [“in” expression [“,” expression]]

注意： exec是一个语法声明，不是一个函数.也就是说和if, for一样

exec 的第一个表达式可以是：

*  代码字符串
*  文件对象
*  代码对象
*  tuple

如果在表达式之后使用 in 选项指定一个 dict，它将作为global和local变量作用域:

>>>  a = 10
>>>  b = 20
>>>  g = {'a':6, 'b':8}
>>>  exec "global a; print a,b" in g  // g同时代表global与local变量作用域
6 8
>>>  exec "print a,b" in g
6 8

如果 in 后详指定两个表达式，它们将分别用作global和local变量作用域:

>>>  a = 1
>>>  b = 2
>>>  c = 3
>>>  g = {'a':10, 'b':20}
>>>  l = {'a':100, 'b ':200, 'c':300}
>>>  exec "global a; print a,b,c" in g,l  // a是从global作用域
10 200 300
>>>  exec "print a,b, c" in g,l  // a是来自local作用域
100 200 300

现在说下tuple的情况，这也是导致很多人误以为 exec 是一个函数的原因( 在3.x版 exec已经是函数.)。如果第一个表达式是tuple:

#  exec(expr, globals)  //它等效与exec expr in globals
#  exec(expr, globals, locals)  //它等效与exec expr in globals,locals

两者比较

  • eval 和 exec 都变成函数.

  • eval 只接受表达式, 而exec就比较丰富了,例如代码段,

  • eval 有返回值,而exec 返回None.
